# Биржа акций 
Проект реализует имитацию биржевых торгов:
администратор настраивает брокеров и запускает симуляцию, брокеры торгуют акциями в реальном времени.

## Архитектура
Проект состоит из трёх независимых приложений:

```
stock-exchange-backend - общий сервер для обоих клиентов
stock-exchange-admin - панель администратора
stock-broker-client - торговый терминал брокера
```

## Технологии
### Backend
- NestJS 11
- TypeScript
- socket.io

### Admin Client
- React 18
- Vite 
- Redux Toolkit 
- Chart.js
- axios 

### Broker Client
- Vue 3 
- Pinia 
- Selenium WebDriver

## Запуск проекта

### Требования
- Node.js 18+
- npm 9+

### 1. Backend

```
cd stock-exchange-backend
npm install
npm run start:dev
# Сервер запустится на http://localhost:3000
```

### 2. Admin Client

```
cd stock-exchange-admin
npm install
npm run dev
# Откроется на http://localhost:5173
```

### 3. Broker Client

```
cd stock-broker-client
npm install
npm run dev
# Откроется на http://localhost:5174
```

## Сценарий использования

1. Запустить backend и оба клиента.
2. Открыть **Admin Client** (`localhost:5173`):
   - На странице **Брокеры** - добавить участников торгов.
   - На странице **Акции** - выбрать тикеры для симуляции, просмотреть графики.
   - На странице **Симуляция** - задать дату начала и скорость, нажать «Начало торгов».
3. Открыть **Broker Client** (`localhost:5174`):
   - Войти под именем одного из созданных брокеров.
   - Наблюдать за изменением котировок в реальном времени.
   - Покупать и продавать акции, следить за балансом и прибылью.
   - Нажать на акцию, чтобы открыть график цены за период симуляции.
4. В Admin Client на странице **Симуляция** можно в любой момент остановить торги.

## E2E-тестирование

Тест находится в `broker-client/e2e.test.cjs`. Требует установленного Firefox и geckodriver.

```
cd stock-broker-client
node tests/e2e.test.js
```

Тест проверяет:
- Вход в систему под именем существующего брокера.
- Покупку 10 акций MSFT - корректное списание с баланса.
- Продажу 5 акций AAPL - корректное зачисление на баланс и уменьшение количества.
